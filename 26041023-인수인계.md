## 7. 실패/문제 발생한 것들 ❌

### ultraplan Windows에서 실패
| 시도 | 결과 | 원인 |
|------|------|------|
| ultraplan 첫 시도 | "Background tasks require a git repository" | git 저장소 미초기화 |
| git init 후 재시도 | 동일 에러 | 커밋이 없었음 |
| 커밋 후 재시도 | 동일 에러 | git user 미설정 |
| user 설정 + 커밋 후 재시도 | 동일 에러 | 원격 저장소 없음 |
| GitHub 연결 + 푸시 후 재시도 | 동일 에러 | **Windows 환경 자체의 호환성 문제** |
| **WSL(Ubuntu)로 전환** | **성공!** | Windows에서 ultraplan 원격 세션 연결 불가 |

### 핵심 교훈
> **ultraplan은 Windows에서 작동하지 않음. WSL(Ubuntu)에서 Claude Code를 실행해야 함.**

### 기타 문제
- `vite-plugin-pwa`가 Vite 8 미지원 → PWA를 수동으로 설정함 (public/sw.js, public/manifest.json)
- PWA 아이콘 이미지(PNG) 미생성 → `public/icons/` 폴더만 있음

## 10. 두 번째 대화 기록 (2026-04-10, WSL 터미널)

> 이 대화는 WSL(Ubuntu)에서 Claude Code 터미널로 진행됨.
> 사용자의 의도는 **울트라플랜에서 작업**하는 것이었으나, 초기에 터미널에서 직접 작업이 진행됨.

### 대화 흐름 요약

#### 1단계: 울트라플랜 승인 → 터미널에서 잘못 실행
- 사용자가 `/ultraplan`으로 프로젝트 세팅을 요청
- 웹(claude.ai)에서 울트라플랜이 승인됨
- **문제**: 울트라플랜 에이전트가 백그라운드에서 작업해야 하는데, 메인 터미널에서 직접 실행해버림
- 결과: 프로젝트 초기 세팅(Vite + React + TS + MUI + Firebase + PWA)은 완료되었으나, 실행 방식이 잘못됨

#### 2단계: 사용자의 울트라플랜 재시도 요청
- 사용자: "울트라 플랜의 기능이 보고 싶은 거야. 터미널에서 한 게 아니라 울트라플랜에서 하고 싶어"
- `/ultraplan` 재실행 → 원격 세션 생성됨
- **문제**: 울트라플랜(웹) 에이전트가 "왜 멈춰있는 거지?"라고 물었을 때, 현재 작업 상황을 모르고 있었음

#### 3단계: 울트라플랜과 터미널 간 컨텍스트 공유 시도
- `PROGRESS.md` 파일을 프로젝트 루트에 생성하여 진행 상황 기록
- **문제**: 울트라플랜(웹)과 터미널은 **파일시스템이 완전히 분리**되어 있음 → PROGRESS.md도 못 찾음
- **핵심 교훈**: 울트라플랜 웹 세션은 원격 환경에서 실행되며, 로컬 WSL 파일시스템과 공유되지 않음

#### 4단계: git push로 코드 공유 시도
- 울트라플랜이 GitHub repo를 클론할 수 있도록 코드를 push하기로 결정
- **발생한 문제들**:

| 순서 | 문제 | 해결 |
|------|------|------|
| 1 | git user identity 미설정 | `git config --global user.email/name` 설정 |
| 2 | `git push` 시 "could not read Username" 에러 | HTTPS credential 문제 |
| 3 | `gh` CLI 미설치 | `gh auth login` 불가 |
| 4 | `git push` 시 비밀번호 입력 안 됨 (사용자가 구글 간편 로그인 사용) | Personal Access Token(PAT) 필요 |
| 5 | WSL 터미널에서 비밀번호 안 보임 | 보안상 정상 동작 (마우스 우클릭으로 붙여넣기) |
| 6 | Username에 한글 인코딩 섞임 (%E3%85%A1) | URL에 토큰 직접 포함하는 방식으로 우회 |

- **최종 성공**: `git remote set-url origin https://masterkota22:TOKEN@github.com/...` 방식으로 push 성공
- push 후 remote URL에서 토큰 제거 완료
- **주의**: 토큰이 대화에 노출됨 → 사용자에게 GitHub에서 토큰 revoke 안내함

#### 5단계: 울트라플랜 재시도
- 새 WSL 창에서 Claude Code 실행 → `/ultraplan` 입력
- 울트라플랜에 입력할 프롬프트 제안:
  ```
  점핑크래쉬_v1.0.md 기획서를 기반으로 프로젝트 초기 세팅이 이미 완료되어 있습니다.
  PROGRESS.md 파일에 완료된 작업 목록이 있으니 먼저 확인해주세요.
  이어서 해야 할 작업: ...
  GitHub repo: https://github.com/masterkota22/jumpingcrash-maxplan
  ```
- **결과**: 울트라플랜 세션이 `error_during_execution`으로 종료됨 (plan approval 전에 에러 발생)

#### 6단계: 울트라플랜(별도 세션)에서 작업 성공 (이 터미널 밖에서)
- 사용자가 별도 창에서 울트라플랜을 성공적으로 실행
- 울트라플랜이 8개 페이지 모두 실제 UI로 구현함 (스텁 → 기능성 UI)
- 추가된 파일/모듈:
  - `src/contexts/AuthContext.tsx` - Firebase Auth 컨텍스트
  - `src/services/firestore.ts` - Firestore CRUD 함수
  - `src/utils/awards.ts` - 시상 등급 판별 유틸
- 이 터미널의 로컬 파일에 system-reminder로 변경 사항이 감지됨

### 이 대화에서의 핵심 교훈

1. **울트라플랜 요청 시 터미널에서 직접 실행하지 말 것** - 울트라플랜은 백그라운드 에이전트가 처리해야 함

4. **GitHub 인증**: 구글 간편 로그인 사용자는 PAT 필요, WSL에서 `credential.helper store` 설정 후 URL에 토큰 포함 방식이 가장 확실
