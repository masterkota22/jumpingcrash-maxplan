## 5. 구현 완료된 기능

### 인증 시스템
- 이메일 로그인/회원가입 (`signInWithEmailAndPassword`, `createUserWithEmailAndPassword`)
- **Google 간편 로그인** (`signInWithPopup` + `GoogleAuthProvider`)
- AuthContext로 전역 인증 상태 관리
- ProtectedRoute로 미인증 접근 차단 (/measure, /records, /profile)
- **이미 로그인 상태면 /login → 홈으로 리다이렉트**

### AI 카메라 줄넘기 측정 (핵심 기능)
- **useCamera**: WebRTC getUserMedia, 전/후면 카메라 전환, 권한 에러 처리
- **usePoseDetector**: MediaPipe PoseLandmarker Lite 모델 CDN 로딩, VIDEO 모드 실시간 감지
- **useJumpCounter**: 점프 감지 알고리즘
  - 골반(23,24), 무릎(25,26), 발목(27,28) 랜드마크 Y좌표 추출
  - kneeAngle = hipKneeDistance / totalLegLength (정규화)
  - 상태 전환: standing → jumping → standing 시 카운트
  - 최소 점프 간격 300ms
- **MeasurePage**: AI 카메라 모드 / 수동 카운트 모드 선택
  - 카메라 미리보기 + 포즈 skeleton 시각화 (canvas 오버레이)
  - 실시간 점프 상태 & 감지 신뢰도 표시
  - 30초 타이머 + 결과 다이얼로그 + 시상 판정

### 기록/순위 시스템
- 기록 저장 (Firestore records 컬렉션)
- 종목별 내 기록 + 최고 기록 + 시상 메달 표시
- 개인 전체 순위 (유저 이름 Firestore 조회)
- 학교 대항전 순위 (상위 3명 평균)

### 대회 시스템
- Competition 타입 (upcoming/active/ended)
- Firestore 연동 (getCompetitions, getCompetition)
- 대회 없으면 빈 상태 UI + 연습 측정 안내

### 학교 검색/등록
- ProfilePage Autocomplete 학교 검색
- Dialog로 새 학교 등록 (이름, 학교급, 지역)

### 전국 학교 데이터
- **334개 seed 데이터** (17개 시도, 초/중/고/대학)
  - `scripts/school-data.json`
  - `npx tsx scripts/seed-schools.ts` 로 Firestore에 업로드
- **나이스 API import 스크립트**
  - `npx tsx scripts/import-neis-schools.ts`
  - 나이스 Open API 키 필요 (https://open.neis.go.kr)
  - 전국 12,000+ 학교 자동 import 가능

### Firebase 배포
- `firebase.json`: Hosting (dist 폴더, SPA rewrite) + Firestore Rules
- `.firebaserc`: 프로젝트 ID (jumping-clash, 사용자가 변경 필요할 수 있음)
- `firestore.rules`: users 본인 쓰기, schools 인증 생성, records 본인 생성, competitions 읽기 전용
- 배포 스크립트:
  - `npm run deploy` — 빌드 + 전체 배포
  - `npm run deploy:hosting` — 빌드 + 호스팅만
  - `npm run deploy:rules` — Firestore 규칙만

### UI/UX
- 반응형 디자인 (모바일 퍼스트)
- BottomNav 5탭: 홈, 측정, 기록, 순위, 대회
- **BottomNav URL 기반 탭 동기화** (뒤로가기 시 불일치 버그 수정)
- 모든 페이지 에러 핸들링 + 로딩 상태
- PWA manifest + 서비스워커 + 아이콘

## 7. 서비스 시작을 위한 사용자 액션

### 필수 (서비스 실행에 반드시 필요)

1. **Firebase 프로젝트 생성**
   - https://console.firebase.google.com 에서 프로젝트 생성
   - Authentication: 이메일/비밀번호 + Google 로그인 활성화
   - Cloud Firestore: 데이터베이스 생성

2. **`.env` 파일 생성** (프로젝트 루트)
   ```
   VITE_FIREBASE_API_KEY=실제값
   VITE_FIREBASE_AUTH_DOMAIN=실제값
   VITE_FIREBASE_PROJECT_ID=실제값
   VITE_FIREBASE_STORAGE_BUCKET=실제값
   VITE_FIREBASE_MESSAGING_SENDER_ID=실제값
   VITE_FIREBASE_APP_ID=실제값
   ```

3. **`.firebaserc` 수정** — 프로젝트 ID를 실제 값으로 변경

4. **Firebase CLI 로그인 + 배포**
   ```bash
   firebase login
   npm run deploy
   ```

5. **학교 데이터 업로드**
   ```bash
   npx tsx scripts/seed-schools.ts
   ```

### 선택

6. **나이스 API 키 발급** (https://open.neis.go.kr)
   - `.env`에 `NEIS_API_KEY=발급받은키` 추가
   - `npx tsx scripts/import-neis-schools.ts` 실행 → 전국 12,000+ 학교 import

7. **실기기 AI 점프 감지 테스트**
   - `src/hooks/useJumpCounter.ts`의 임계값 조정 가능:
     - `JUMPING_THRESHOLD = 0.3` (점프 판정)
     - `SQUATTING_THRESHOLD = 0.45` (구부림 판정)
     - `MIN_JUMP_INTERVAL_MS = 300` (최소 점프 간격)
