## 1. 이번 세션 배경

이전 세션(4차, 2026-04-16)까지 AI 카메라 측정 기능이 "코드상" 완성되어 있었으나, 실기기 테스트 시 **카메라 기능이 실질적으로 동작하지 않음**을 사용자가 발견:

### 사용자가 보고한 증상
1. **1차 (카메라 자체 불능):** 카메라 권한은 허용되는데 `<video>` 영역이 하얀 화면으로만 보임. AI 모델 "준비 완료" 메시지는 뜨지만 실제 감지는 0. 시작! 누르면 타이머만 돌고 카운트는 안 올라감.
2. **2차 (감지 노이즈):** 1차 수정 후 영상은 보이는데, 사람이 가만히 있어도 초록색 뼈대가 랜덤하게 흔들림. 얼굴도 감지 안 됨.
3. **3차 (스마트폰 테스트):** 2차 수정 배포 후에도 스마트폰(아마 셀피)에서 뼈대가 전혀 안 보임.

## 2. 이번 세션에서 수행한 수정

### 2-1. 스트림 바인딩 버그 (commit `5b7413c`)

**파일:** `src/hooks/useCamera.ts`, `src/pages/MeasurePage.tsx`

**원인 트레이스:**
- `useCamera.start()` 가 `status='starting'` 상태에서 `videoRef.current.srcObject = stream` 을 시도했는데, `<video>` 엘리먼트는 `status==='active'` 일 때만 렌더되므로 `ref.current` 가 **null** — 스트림이 어디에도 연결 안 됨
- `ready → measuring` phase 전환 시 두 개의 별도 `<video>` JSX 가 리마운트되며 이전 엘리먼트에만 붙어있던 `srcObject` 가 유실
- `toggleFacing` → `setTimeout(start, 300)` 은 stale closure 로 이전 facing 값 사용

**수정 내용:**
- `useCamera`: `videoRef` 제거, `stream: MediaStream | null` 을 state 로 노출. `start()` 는 스트림 획득만 책임. DOM 은 소비자 컴포넌트가 관리
- `MeasurePage`: 로컬 `videoRef` 도입. `useEffect([camera.stream, phase])` 로 엘리먼트가 확실히 마운트된 뒤 srcObject/play 수행. phase 전환 리마운트 시에도 재실행되어 새 엘리먼트에 자동 재연결
- `toggleFacing` 은 facing 갱신 후 활성 스트림이 있으면 즉시 `start()` 재호출로 단순화

### 2-2. 포즈 감지 노이즈 + 전신 스켈레톤 (commit `81c9474`)

**파일:** `src/hooks/usePoseDetector.ts`, `src/hooks/useCamera.ts`, `src/pages/MeasurePage.tsx`

**원인 4가지:**
1. MediaPipe WASM CDN 을 `@latest` 로 로드 → 설치된 npm 패키지 `0.10.34` 와 버전 불일치 시 랜드마크가 노이즈가 됨
2. `requestAnimationFrame` 이 동일 ms 에 2회 콜되면 동일 timestamp 가 들어가 VIDEO 모드의 monotonic-increase 규칙을 위반 → 감지 실패
3. 뼈대 렌더링이 하체(landmark 23~28) 6개 포인트만 그려서 얼굴·팔·몸통 감지 여부 확인 불가. 셀피 자세면 다리가 프레임 밖이라 **뼈대 0개가 정상**처럼 보임
4. 카메라 해상도 640×480 은 전신 포즈 감지에 빈약

**수정 내용:**
- `usePoseDetector`:
  - WASM CDN 버전 `0.10.34` 고정 (`const MEDIAPIPE_VERSION = '0.10.34'`)
  - 모델 URL `/latest/` → `/1/` 로 고정
  - GPU delegate 실패 시 CPU 로 자동 폴백 (try/catch 2단)
  - `lastTimestampRef` 로 timestamp 중복 방지 (`<=` 이면 `+1` 강제)
  - 신뢰도 임계값 `0.5 → 0.6`
- `MeasurePage`:
  - **전신 33 키포인트 + 연결선 렌더**
    - 얼굴(0~10): 파랑 `#40C4FF`
    - 팔(11~22): 노랑 `#FFC400`
    - 몸통(어깨·엉덩이): 흰색 `#FFFFFF`
    - 다리(23~32): 초록 `#00E676`
  - `visibility < 0.5` 포인트는 렌더 생략 → 흔들림 제거
- `useCamera`: `1280×720, 30fps` 로 상향

## 4. ⚠️ 미해결 이슈 — 로컬 / 배포 버전 불일치

### 현상
- 원격 세션이 push 한 `45e5b2f` / `81c9474` / `5b7413c` 와는 별개로, 사용자 로컬 세션이 동일한 수정을 독립 커밋(`4efc42d`, `68faf9a` 등)했고 이 상태에서 Firebase Hosting 배포가 이루어짐
- 결과: **배포된 `https://jumping-crash.web.app` 은 스트림 바인딩 수정만 포함하고, 포즈 감지 개선(전신 스켈레톤·WASM 버전 고정·CPU 폴백 등)이 반영되지 않음**
- 스마트폰 테스트 시 "뼈대 안 보임" 증상은 이 때문

### 해결 절차 (사용자가 로컬 머신에서 수행)

```bash
cd c:\taeple\jumping-crash

# 1) 원격 최신 가져오기
git fetch origin
git log origin/master --oneline -5
# → 45e5b2f, 81c9474, 5b7413c 가 보여야 함

# 2) 로컬에 고유 커밋이 없는지 확인 (중복 커밋 외에)
git log master --oneline -10
git log master ^origin/master --oneline  # 로컬에만 있는 커밋

# 3) 로컬을 원격 기준으로 재설정 (로컬 중복 커밋 폐기)
git checkout master
git reset --hard origin/master

# 4) 의존성 동기화 + 빌드 + 배포
npm install
npm run build
firebase deploy --only hosting
```

> **⚠️ `git reset --hard` 주의:** 로컬 커밋을 되돌릴 수 없게 버립니다. Step 2 에서 중복 커밋(`4efc42d`, `68faf9a`) 외에 의미 있는 로컬 작업이 있다면 `rebase` 로 처리. 확실치 않으면 `git branch backup-local master` 로 백업 브랜치부터 만들 것.

## 5. 배포 후 검증 체크리스트

### 데스크톱 (Chrome/Edge)
- [ ] 브라우저 강력 새로고침 (Ctrl+Shift+R)
- [ ] `/measure` → AI 카메라 → 카메라 시작 → 영상이 즉시 뜸
- [ ] AI 모델 "준비 완료" → 시작! → 측정 phase 에도 영상 유지
- [ ] 전신을 프레임에 담았을 때 얼굴(파랑)·팔(노랑)·몸통(흰)·다리(초록) 뼈대가 몸을 따라 움직임
- [ ] 가만히 있을 때 뼈대도 가만히 있음 (랜덤 흔들림 X)
- [ ] 제자리 점프 시 `점프! ↔ 착지` 칩 전환 + 카운트 증가
- [ ] 전/후면 카메라 버튼 1회 클릭으로 전환 성공

### 스마트폰 (iOS Safari / Android Chrome)
- [ ] 시크릿 탭 또는 사이트 데이터 삭제 후 접속
- [ ] 얼굴만 보이는 상태에서도 파란 얼굴 뼈대 + 흰 몸통선 + 노란 팔선 표시됨
- [ ] 전신 프레임에서 초록 다리선까지 표시
- [ ] 측정 중 화면이 끊기거나 페이지가 다시 로드되지 않음

### 문제 시 디버깅
- 스마트폰: Chrome DevTools 원격 디버깅 (`chrome://inspect#devices`) 로 콘솔 확인
- iOS Safari: Mac Safari 개발자 메뉴 → 디바이스 선택 → 웹 인스펙터
- 주요 확인 항목:
  - MediaPipe WASM 로딩 실패 (네트워크 탭 `@mediapipe/tasks-vision@0.10.34/wasm`)
  - GPU delegate 경고 로그 (`GPU delegate failed, falling back to CPU`)
  - `PoseLandmarker load error:` 로그
