# 점핑크래쉬 (Jumping Crash) v1.0

> 전국 학교 대항 줄넘기 대결 플랫폼
> 작성일: 2026-04-10

---

## 1. 프로젝트 개요

**점핑크래쉬**는 휴대폰 카메라를 이용해 줄넘기 횟수를 자동 측정하고,
전국 초/중/고/대학교 학생들이 학교 대항으로 줄넘기 대결을 할 수 있는 웹 플랫폼이다.

### 핵심 가치
- **공정한 자동 측정**: 카메라 AI로 줄넘기 횟수를 정확히 측정
- **학교 대항 경쟁**: 소속 학교 기반으로 순위 경쟁
- **접근성**: 별도 장비 없이 휴대폰 하나로 참여 가능

---

## 2. v1.0 범위 (MVP)

### 대회 종목
v1.0에서는 **스피드 30초 줄넘기** 3종목만 구현한다:

| 종목 | 설명 |
|------|------|
| 모아뛰기 | 두 발을 모아서 뛰는 기본 줄넘기 |
| 번갈아뛰기 | 양 발을 번갈아 뛰는 줄넘기 (구보뛰기) |
| 이중뛰기 | 한 번 점프에 줄을 두 번 돌리는 줄넘기 |

> 추후 다른 종목(3중뛰기, 크로스, 단체줄넘기 등) 추가 예정

### 경기 방식
- **30초** 동안 줄넘기를 뛰고, 그 안에서 성공한 횟수를 기록
- 줄에 걸리거나 줄 인식 실패 시 카운트하지 않음

## 3. 카메라 AI 측정 - 핵심 기술 과제

### 3.1 공통 인식 요소
- **사람 인식**: 전신 포즈 감지 (발, 무릎, 엉덩이, 어깨 등 키포인트)
- **줄 인식**: 줄넘기 줄의 위치/궤적 추적
- **바닥 기준선**: 발이 바닥에서 떨어지는 순간 감지

### 3.2 종목별 측정 로직 (핵심 고민 포인트)

#### 모아뛰기 측정
- **정의**: 두 발이 동시에 바닥에서 떨어졌다가 착지, 그 사이에 줄이 발 아래를 1회 통과
- **카운트 조건**:
  1. 두 발이 동시에 바닥에서 떨어짐 (점프 감지)
  2. 점프 중 줄이 머리 위 → 발 아래로 1회 회전 완료
  3. 두 발이 다시 바닥에 착지
- **미카운트 조건**:
  - 줄이 발에 걸림 (줄 궤적 중단/급정지)
  - 줄이 완전히 1회전하지 못함
  - 점프 없이 줄만 돌린 경우

#### 번갈아뛰기 측정
- **정의**: 왼발/오른발이 번갈아 착지, 줄이 매 스텝마다 발 아래를 통과
- **카운트 조건**:
  1. 한쪽 발이 올라가고 반대쪽 발이 착지 (스텝 감지)
  2. 스텝 전환 사이에 줄이 발 아래를 1회 통과
  3. 발이 실제로 번갈아 교체됨
- **미카운트 조건**:
  - 줄에 걸림
  - 같은 발로 연속 2번 착지 (모아뛰기와 혼동 방지)
  - 줄 회전 없이 발만 번갈아 움직인 경우
- **주의**: 모아뛰기보다 동작이 빠르고, 발 교체 시점 감지가 정확해야 함

#### 이중뛰기 측정
- **정의**: 1번 점프 동안 줄이 발 아래를 2회 통과
- **카운트 조건**:
  1. 점프 감지 (발이 바닥에서 떨어짐)
  2. 공중에 있는 동안 줄이 2회 완전 회전
  3. 착지
- **미카운트 조건**:
  - 줄이 1회만 회전 (일반 모아뛰기로 처리하지 않음, 이중뛰기 모드에서는 미카운트)
  - 줄에 걸림
  - 줄이 2회전을 완료하기 전에 착지
- **주의**: 줄 회전 속도가 매우 빠름 → 고프레임 처리 필요 (최소 60fps 이상 권장)

### 3.3 기술적 도전과 고민사항

| 과제 | 설명 | 우선순위 |
|------|------|----------|
| 줄 인식 정확도 | 얇은 줄넘기 줄을 카메라로 인식하기 어려움 | 최상 |
| 줄 걸림 판별 | 줄에 걸렸는지 vs 정상 회전인지 구분 | 최상 |
| 프레임 레이트 | 이중뛰기는 줄 속도가 빠라 고fps 필요 | 상 |
| 촬영 환경 | 배경, 조명, 카메라 각도에 따른 인식률 변화 | 상 |
| 모바일 성능 | 실시간 AI 처리를 모바일에서 수행 가능한가 | 상 |
| 종목 자동 구분 | 모아뛰기/번갈아뛰기 혼동 없이 정확히 구분 | 중 |

### 3.4 촬영 가이드라인 (예상)
- 카메라 위치: 측면에서 전신이 보이도록 촬영
- 거리: 사람의 전신 + 줄 궤적이 화면 안에 들어오는 거리
- 배경: 단색 배경 권장 (줄 인식률 향상)
- 조명: 충분한 밝기, 역광 피하기

## 5. 순위 체계

### 5.1 개인전 순위
- 종목별로 모든 참가자의 기록을 개수 기준 내림차순 정렬
- 개인에게는 시상 기준표에 따라 **금/은/동** 등급 부여
- 학년/부문별로 구분하여 순위 매김

### 5.2 학교 대항전 순위

#### 산출 방식
1. 각 학교의 해당 학년 학생들의 기록을 수집
2. **상위 3명의 기록만 평균에 포함** (핵심 규칙)
3. 순위에 들지 못한(4위 이하) 학생의 기록은 평균에서 제외
4. 평균값 기준으로 학교 순위를 매김

#### 예시

**A학교 1학년** (30초 모아뛰기)
| 학생 | 기록 | 평균 포함 |
|------|------|-----------|
| ㄴ학생 | 70개 | O (1위) |
| ㄷ학생 | 67개 | O (2위) |
| ㄱ학생 | 62개 | O (3위) |
| **합계** | **199개** | **평균 66.33개** |

**B학교 1학년** (30초 모아뛰기)
| 학생 | 기록 | 평균 포함 |
|------|------|-----------|
| ㄷ학생 | 85개 | O (1위) |
| ㄴ학생 | 60개 | O (2위) |
| ㄱ학생 | 59개 | O (3위) |
| **합계** | **204개** | **평균 68.00개** |

**개인전 순위**: 1위 B학교 ㄷ학생(85개) → 2위 A학교 ㄴ학생(70개) → 3위 A학교 ㄷ학생(67개)

**학교 대항전 순위**: 1위 B학교(평균 68.00) → 2위 A학교(평균 66.33)

#### 규칙 정리
- 학교 대항전 평균은 **상위 3명**까지만 포함
- 4명 이상 참가해도 상위 3명만 학교 점수에 반영
- 4위 이하 학생은 **개인전 기록에만** 남음
- 학년별로 별도 순위 산출

## 6. 사용자 구조

### 소속 체계
```
학교급 (초등/중등/고등/대학)
  └── 학교명
        └── 학년
              └── 학생 (개인 계정)
```

### 필요 정보
- 학생: 이름, 학교, 학년, 반(선택)
- 학교: 학교명, 학교급, 지역
- 학교 인증 방식은 추후 결정 (교사 승인, 학교 코드 등)

## 7. 주요 화면 구성 (예상)

| 화면 | 설명 |
|------|------|

| 학교 순위 | 학교별 평균 기록, 학년별 순위 |
| 개인 순위 | 전체 개인 기록 순위 |
| 대회 | 대회 목록, 참가 신청, 대회 결과 |

## 8. 현재 고민 & 결정 필요 사항

### 기술 고민
- [ ] 줄 인식을 어떤 AI 모델/라이브러리로 할 것인가? (MediaPipe, TensorFlow.js, 커스텀 모델 등)
- [ ] 줄이 너무 얇아서 인식이 안 될 경우 대안은? (줄에 색깔 테이프? 특수 줄?)
- [ ] 모바일 브라우저에서 실시간 처리가 가능한가? (네이티브 앱이 필요할 수도)
- [ ] 이중뛰기의 빠른 줄 속도를 모바일 카메라 fps로 감당 가능한가?

### 서비스 고민
- [ ] 학교 인증은 어떻게 할 것인가? (가짜 학교 소속 방지)
- [ ] 부정행위 방지 (녹화 조작, 타인 영상 제출 등)
- [ ] 대회 기간 설정 (상시 vs 기간 한정)
- [ ] 참가 자격 검증 방법
- [ ] 3명 미만인 학교의 학교 대항전 참가 가능 여부

### 비즈니스 고민
- [ ] K-ROPE 협회와의 연계/공인 대회 가능성
- [ ] 수익 모델 (대회 참가비, 학교 구독 등)

## 10. 로드맵

### Phase 1 - MVP (현재 계획)
- 30초 스피드 줄넘기 3종목 (모아/번갈아/이중)
- 카메라 AI 자동 측정
- 개인 기록 & 등급
- 학교 대항전 순위 (상위 3명 평균)
- 개인전 순위

### Phase 2 - 확장
- 종목 추가 (1분 스피드, 3중뛰기, 크로스 등)
- 단체 줄넘기 종목
- 대회 생성/관리 기능
- 지역별 순위

### Phase 3 - 고도화
- 실시간 대결 (1:1 라이브 대결)
- 리플레이/하이라이트 영상
- 코칭 기능 (AI가 자세 분석)
- 다른 스포츠 확장 (태권플레이, 사커플레이 등 올플레이 생태계)
